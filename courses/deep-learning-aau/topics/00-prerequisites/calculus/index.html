Calculus.

Calculus is the mathematical engine that drives modern deep learning. From optimizing loss functions using gradient descent to backpropagating errors through deep networks, a firm grasp of derivatives, gradients, the chain rule, and Taylor approximations is essential.

What this note covers

  • The 1D derivative and its geometric intuition
  • Partial derivatives and gradients in multiple dimensions
  • The Chain Rule and backpropagation foundations
  • Taylor expansions and their role in optimization
  • A lightweight automatic differentiation engine in Python

For a detailed mathematical treatment, see Chapter 5 of Mathematics for Machine Learning.

The derivative (1D)

At its core, a derivative represents the instantaneous rate of change of a function. Geometrically, the derivative $f'(x)$ of a function $f(x)$ at a point $x$ is the slope of the tangent line to the curve at that point.

Mathematically, we define it as the limit of the average rate of change as the interval shrinks to zero:

$$ f'(x) = \lim_{h \to 0} \frac{f(x + h) - f(x)}{h} $$

Let's look at the quadratic function $f(x) = x^2 - 4x + 5$. Using standard calculus rules, its derivative is given by $f'(x) = 2x - 4$. Below is an interactive plot of this function. You can use the slider to change the point of tangency and observe how the slope of the tangent line changes.

In machine learning, we look for points where the derivative is zero (critical points) to find local minima of loss functions, just like the vertex of the parabola above.

Gradients and partial derivatives (N-D)

Deep learning rarely involves functions of a single variable. When a function $f(\mathbf{x})$ depends on a vector of inputs $\mathbf{x} = [x_1, x_2, \dots, x_d]^\top$, we compute the rate of change with respect to one variable at a time while holding all others constant. This is the partial derivative, denoted by $\frac{\partial f}{\partial x_i}$.

For example, if we have the function $f(x, y) = x^2 + 2y^2$, the partial derivatives are:

$$ \frac{\partial f}{\partial x} = 2x \quad \text{and} \quad \frac{\partial f}{\partial y} = 4y $$

The gradient of a function, denoted by $\nabla f(\mathbf{x})$, is a vector containing all its partial derivatives:

$$ \nabla f(\mathbf{x}) = \begin{bmatrix} \frac{\partial f}{\partial x_1} \\ \frac{\partial f}{\partial x_2} \\ \vdots \\ \frac{\partial f}{\partial x_d} \end{bmatrix} $$

important

The gradient has a fundamental geometric property: the gradient vector points in the direction of the steepest ascent of the function at that point. Its magnitude indicates the rate of increase. Consequently, moving in the opposite direction, $-\nabla f(\mathbf{x})$, gives the direction of steepest descent, which is the basis for gradient descent optimization.

The Chain Rule

In deep learning, neural networks are constructed by stacking layers. Mathematically, this means a neural network is a giant composite function. If we have a chain of operations $y = f(u)$ and $u = g(x)$, the derivative of $y$ with respect to $x$ is given by the chain rule:

$$ \frac{dy}{dx} = \frac{dy}{du} \frac{du}{dx} $$

For multi-variable functions, the chain rule scales to sum over all paths. If $z = f(u, v)$ and $u = g(x)$, $v = h(x)$, then:

$$ \frac{dz}{dx} = \frac{\partial z}{\partial u}\frac{du}{dx} + \frac{\partial z}{\partial v}\frac{dv}{dx} $$

Applying the Chain Rule to a Neuron

Let's look at a concrete deep learning example. Consider a single neuron with input $x$, weight $w$, bias $b$, and sigmoid activation function $\sigma(z) = \frac{1}{1 + e^{-z}}$. The output $a$ is:

$$ a = \sigma(z) \quad \text{where} \quad z = wx + b $$

We want to find how the output changes with respect to the weight $w$. By the chain rule:

$$ \frac{\partial a}{\partial w} = \frac{\partial a}{\partial z} \frac{\partial z}{\partial w} $$

First, we compute the derivative of the sigmoid function, which has the elegant property $\sigma'(z) = \sigma(z)(1 - \sigma(z)) = a(1 - a)$. Then, we compute the derivative of $z$ with respect to $w$, which is simply $x$. Substituting these back:

$$ \frac{\partial a}{\partial w} = a(1 - a) \cdot x $$

This simple calculation is the core building block of the backpropagation algorithm used to train deep networks.

Taylor Expansions

A Taylor series approximates a complex differentiable function locally around a point $x_0$ using a polynomial. The first-order Taylor approximation (also known as linearization) is given by:

$$ f(x) \approx f(x_0) + f'(x_0)(x - x_0) $$

In multiple dimensions, this becomes:

$$ f(\mathbf{x}) \approx f(\mathbf{x}_0) + \nabla f(\mathbf{x}_0)^\top (\mathbf{x} - \mathbf{x}_0) $$

This approximation shows why Gradient Descent works. If we update our position by moving a small step $\eta > 0$ in the negative gradient direction, i.e., setting $\mathbf{x} - \mathbf{x}_0 = -\eta \nabla f(\mathbf{x}_0)$, we get:

$$ f(\mathbf{x}_0 - \eta \nabla f(\mathbf{x}_0)) \approx f(\mathbf{x}_0) - \eta \|\nabla f(\mathbf{x}_0)\|^2 < f(\mathbf{x}_0) $$

Since the squared norm $\|\nabla f(\mathbf{x}_0)\|^2$ is strictly positive (unless the gradient is zero), this guarantees that our function value decreases for a sufficiently small step size $\eta$.

Implementation: Automatic Differentiation

Modern deep learning frameworks like PyTorch and JAX use automatic differentiation (autodiff) to compute gradients. Below is a complete implementation of a lightweight autodiff engine from scratch in Python. It builds a directed acyclic graph (DAG) of operations and performs a backward pass using the chain rule.

First, let's set up a virtual environment and install dependencies:

uv venv --python 3.13
# No external libraries are needed for this core implementation!

We define a class Var that wraps a value and tracks operations:

class Var:
    def __init__(self, value, creator=None, op=""):
        self.value = value
        self.grad = 0.0
        self.creator = creator
        self.op = op
        self._backward = lambda: None

    def __add__(self, other):
        other = other if isinstance(other, Var) else Var(other)
        out = Var(self.value + other.value, creator=(self, other), op="+")
        
        def _backward():
            self.grad += out.grad
            other.grad += out.grad
        out._backward = _backward
        return out

    def __radd__(self, other):
        return self.__add__(other)

    def __mul__(self, other):
        other = other if isinstance(other, Var) else Var(other)
        out = Var(self.value * other.value, creator=(self, other), op="*")
        
        def _backward():
            self.grad += other.value * out.grad
            other.grad += self.value * out.grad
        out._backward = _backward
        return out

    def __rmul__(self, other):
        return self.__mul__(other)

    def backward(self):
        # Topological sort to order the nodes for backpropagation
        topo = []
        visited = set()
        
        def build_topo(v):
            if v not in visited:
                visited.add(v)
                if v.creator:
                    for child in v.creator:
                        build_topo(child)
                topo.append(v)
                
        build_topo(self)
        
        # Set seed gradient to 1.0 at the output
        self.grad = 1.0
        for v in reversed(topo):
            v._backward()

    def __repr__(self):
        return f"Var(value={self.value:.4f}, grad={self.grad:.4f})"

Let's verify this engine by computing derivatives of a simple function $y = a \cdot b + c$ with respect to its inputs at $a=3, b=2, c=1$:

if __name__ == "__main__":
    a = Var(3.0)
    b = Var(2.0)
    c = Var(1.0)
    
    # Forward pass: y = a * b + c
    y = a * b + c
    
    # Backward pass
    y.backward()
    
    print(f"y: {y.value} (Expected: 7.0)")
    print(f"dy/da (gradient of a): {a.grad} (Expected: 2.0)")
    print(f"dy/db (gradient of b): {b.grad} (Expected: 3.0)")
    print(f"dy/dc (gradient of c): {c.grad} (Expected: 1.0)")

This prints exactly the values derived analytically. By building a graph of operations and passing gradients backward, the code implements the chain rule automatically!
