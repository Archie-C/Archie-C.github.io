Multiple Input Dimensions.

It is very unusual to only have one input feature affect the target. Welcome to the multivariate case, where we have multiple input features of varying importance.

What this note covers

  • The model
  • The objective function
  • Implementation details

For a more formal definition, see Deep learning: Foundations and Concepts

The model

So far we have only looked at the case where each data point has a single feature (i.e. it is just one number). But what if we have multiple features? For example, the price of a house, we might have features like the size of the house, the number of bedrooms, the location, etc. In this case, we can extend our linear regression model to handle multiple input dimensions:

$$\hat{y} = w_0 + w_1 x_1 + w_2 x_2 + ... + w_n x_n$$

Conceptually, it is the same. Each feature has a corresponding weight that needs to be learned and a bias term $w_0$ that allows us to shift the line up or down.

Even though we can't easily visualise this in more than 3 dimensions, we can still find the optimal parameters using the same least squares function as before. However, the 2D formula was long enough, so if we try to write them out as before we would have $n$ simultaneous equations and it would be a nightmare to solve. Luckily, we can use some linear algebra to write this in a much more compact form.

We can define the input vector and the weight vector as follows:

$$\mathbf{x} = \begin{bmatrix} 1 \\ x_1 \\ x_2 \\ \vdots \\ x_n \end{bmatrix}, \quad \mathbf{\theta} = \begin{bmatrix} w_0 \\ w_1 \\ w_2 \\ \vdots \\ w_n \end{bmatrix}$$

important

The vector $\mathbf{x}$ includes a 1 in the first position to account for the bias term $w_0$. This means that both vectors are now length $n+1$ (including the bias term).

We transition to the more typical notiation of $\mathbf{\theta}$ for the weight vector. Then the model can be written in vector form as:

$$\hat{y}(\mathbf{x}, \mathbf{\theta}) = \mathbf{\theta}^\top \mathbf{x}$$

The objective function

This gives quite a concise least squares formulation:

$$\mathcal{L}(\mathbf{\theta}) = \sum_{i=1}^M (y_i - \mathbf{\theta}^\top \mathbf{x}_i)^2$$

But we can make it even more compact by using matrix notation. For multiple data points, we can define a design matrix. Which is essentially just the input vectors stacked on top of each other:

$$ \mathbf{X} = \begin{bmatrix} 1 & x_{11} & x_{12} & \cdots & x_{1n} \\ 1 & x_{21} & x_{22} & \cdots & x_{2n} \\ \vdots & \vdots & \vdots & \ddots & \vdots \\ 1 & x_{M1} & x_{M2} & \cdots & x_{Mn} \end{bmatrix}, \quad \mathbf{y} = \begin{bmatrix} y_1 \\ y_2 \\ \vdots \\ y_M \end{bmatrix} $$

Here, $M$ is the number of data points and $n$ is the number of input features. Using this notation, and the same weight vector as before, we can write the objective function even more concisely:

$$\mathcal{L}(\mathbf{\theta}) = \|\mathbf{y} - \mathbf{X} \mathbf{\theta}\|^2$$

Where $\|\cdot\|^2$ denotes the L2 norm, which is the sum of the squares of the elements of a vector.

Before we find the minimum of this function we need to do some rearrangement. The first thing to note is that the above is just equal to the transpose of the difference multiplied by the difference:

$$\mathcal{L}(\mathbf{\theta}) = (\mathbf{y} - \mathbf{X} \mathbf{\theta})^\top (\mathbf{y} - \mathbf{X} \mathbf{\theta})$$

Which we can expand out to:

$$\mathcal{L}(\mathbf{\theta}) = \mathbf{y}^\top \mathbf{y} - 2 \mathbf{y}^\top \mathbf{X} \mathbf{\theta} + \mathbf{\theta}^\top \mathbf{X}^\top \mathbf{X} \mathbf{\theta}$$

Now we can take the gradient of this with respect to $\mathbf{\theta}$ and set it equal to zero to find the minimum:

$$\nabla_\mathbf{\theta} \mathcal{L}(\mathbf{\theta}) = -2 \mathbf{X}^\top \mathbf{y} + 2 \mathbf{X}^\top \mathbf{X} \mathbf{\theta} = 0$$

Again, I omit the detailed derivation here as I emplore you to work through it yourself. From this, it is actually trivial to rearrange for $\mathbf{\theta}$:

$$\mathbf{\theta} = (\mathbf{X}^\top \mathbf{X})^{-1} \mathbf{X}^\top \mathbf{y}$$

Implementation

This section builds on the base class Regression which we implemented in the Linear Regression chapter. If you haven't done that, do that and then come back to this section.

So first, we need a class that inherits from Regression:

class MultivariateLinearRegression(Regression):

Then we can implement the constructor which only needs one variable this time, the vector $\mathbf{\theta}$:

def __init__(self):
    self.theta = None

For the fit method, we will assume that it is only the features matrix $\mathbf{X}$ and the target vector $\mathbf{y}$ being passed in. Which means we need to add a column of ones to the design matrix to account for the bias term and then we can use the closed-form solution we derived above to find the optimal weights.

def fit(self, X, y):
    n_samples, n_features = X.shape
    bias = np.ones((n_samples, 1))
    X_design = np.hstack((bias, X))
    self.theta = np.linalg.inv(X_design.T @ X_design) @ X_design.T @ y

Finally, the predict method. Again, we need to add a column of ones to the input matrix before making predictions.

def predict(self, X):
    n_samples, n_features = X.shape
    bias = np.ones((n_samples, 1))
    X_design = np.hstack((bias, X))
    return X_design @ self.theta
