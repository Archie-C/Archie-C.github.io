Non-Linear Regression.

What if the relationships aren't linear?

What this note covers

  • The premise
  • Basis functions
  • The model
  • The optimal weights
  • Implementation details

For a more formal definition, see Deep learning: Foundations and Concepts

The premise

In all the previous sections we were looking at modelling a linear relationship between input variables and target variables. Now we extend this idea to consider non-linear relationships.

The way we do that is by transforming the input features into a higher dimensional space then fitting a linear model in that space. This is where basis functions step in.

Basis functions

A basis function is a function that maps the input features into some other space. For example, we could use a polynomial basis function, which takes the input features and raise them to some power. For example, if we have a single input feature x, we could use the following basis function:

$$ \phi(x) = \begin{bmatrix} 1 \\ x \\ x^2 \\ \vdots \\ x^d \end{bmatrix} $$

What this does is returns a vector of transformed features up to degree $d$. One can imagine that this model will be much more expressive than the simple linear case. This allows us to capture non-linear relationships between the input and the target variables.

There are many different types of basis functions:

  • Polynomial basis functions

    $\phi(x) = [ 1, x, x^2, \cdots, x^d ]$

  • Fourier basis functions

    $\phi(x) = [ 1, \cos(x), \sin(x), \cos(2x), \sin(2x), \cdots ]$

  • Sinusoidal basis functions

    $\phi(x) = [ \sin(\gamma (x - c_1)), \sin(\gamma (x - c_2)), \cdots ]$

  • Gaussian basis functions

    $\phi(x) = [ \exp(-\gamma (x - c_1)^2), \exp(-\gamma (x - c_2)^2), \cdots ]$

  • Sigmoidal basis functions

    $\phi(x) = [ \sigma(\gamma (x - c_1)), \sigma(\gamma (x - c_2)), \cdots ]$

to list a few...

In practice, the choice of the basis function is crucial for the performance of the model.

The model

We will use a second order polynomial basis function for the rest of the example for simplicity. The steps are the same with any basis function.

First, lets extend it to multi-input and multi-output scenarios. In this case our basis function looks something like this:

$$ \phi(\mathbf{x}) = [ 1, x_1, x_2, \cdots, x_n, x_1^2, x_2^2, \cdots, x_n^2 ] $$

important

We choose to omit the cross terms here but they can be included if needed. The above does not fully define a quadratic surface but is simpler. Instead it defines separable quadratic surfaces.

Like before, we can define the design matrix and target matrix in this new feature space and apply the same least squares method. The design matrix is given by:

$$ \mathbf{\Phi} = \begin{bmatrix} 1 & x_1^{(1)} & x_2^{(1)} & \cdots & (x_1^{(1)})^2 & (x_2^{(1)})^2 & \cdots \\ 1 & x_1^{(2)} & x_2^{(2)} & \cdots & (x_1^{(2)})^2 & (x_2^{(2)})^2 & \cdots \\ \vdots & \vdots & \vdots & \ddots & \vdots & \vdots & \ddots \\ 1 & x_1^{(m)} & x_2^{(m)} & \cdots & (x_1^{(m)})^2 & (x_2^{(m)})^2 & \cdots \\ \end{bmatrix}$$

here we use $x^{(m)}$ to denote the $m$-th training example and $x_n$ to denote the $n$-th input feature.

Now that is very long, so we typically use a more compact notation like this:

$$ \mathbf{\Phi} = \begin{bmatrix} \phi(\mathbf{x}^{(1)}) \\ \phi(\mathbf{x}^{(2)}) \\ \vdots \\ \phi(\mathbf{x}^{(m)}) \end{bmatrix} $$

The target matrix is defined as before with $l$ being the number of output dimensions. This is pretty much the same as the multiple output case, however now that we have transformed our input space into more dimensions, the weight matrix also needs those dimensions.

In this example, we have $1$ bias term, then $n$ linear terms, and $n$ quadratic terms, for a total of $1 + n + n = 1 + 2n$ features. So the weight matrix becomes $\mathbf{\Theta} \in \mathbb{R}^{(1 + 2n) \times l}$. Which, conveniently, is the same shape as the design matrix transposed times the target matrix, which is what we need for the closed form solution.

The optimal weights

With all that, we can apply the same least squares method to find the optimal weights:

$$ \mathbf{\Theta} = (\mathbf{\Phi}^\top \mathbf{\Phi})^{-1} \mathbf{\Phi}^\top \mathbf{Y} $$

Implementation

The implementation is straightforward, the only challenge is defining the basis functions and constructing the design matrix. We can set up the class and call the constructor the same as before:

class NonLinearRegression:
    def __init__(self, degree: int = 2):
        self.degree = degree
        self.theta = None

The only difference from the linear case is the addition of the `degree` parameter to specify the polynomial degree.

Then we need to implement the fit method in which we will construct the design matrix and solve for $\mathbf{\Theta}$:

def fit(self, X, Y):
    design_matrix = self._construct_design_matrix(X)
    self.theta = np.linalg.inv(design_matrix.T @ design_matrix) @ design_matrix.T @ Y

Here we call a helper method to construct the matrix, as we will also need to construct it for prediction. The helper method we can define as follows:

def _construct_design_matrix(self, X):
    n_samples, _ = X.shape
    bias = np.ones((n_samples, 1))
    X_poly = bias
    for d in range(1, self.degree + 1):
        X_poly = np.hstack((X_poly, X ** d))
    return X_poly

So we first define a column for the bias term, then append the polynomial features for each degree iteratively and return the resulting matrix. Now, finally we can write the predict method:

def predict(self, X):
    design_matrix = self._construct_design_matrix(X)
    return design_matrix @ self.theta

This is just one way to implement the non-linear regression model. You can modify the basis functions to include cross terms or use different types. The rest of the implementation remains the same.
